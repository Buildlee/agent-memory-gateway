# Product

## Register

product

## Users

这套系统面向需要让 Codex、Hermes Agent、OpenClaw 等多个 Agent 共用长期记忆的个人用户与小型团队。日常管理者不一定熟悉容器、数据库或命令行；进入管理页时，他们通常只想确认系统是否正常、哪些记忆需要处理、设备拥有哪些权限，以及某次操作来自哪里。

## Product Purpose

Agent Memory Gateway 把共享记忆从单个 Agent 的本地能力提升为独立、可审计的服务。系统应让用户在一个受保护的管理入口中完成记忆检索、审核、运行检查、设备授权和来源追踪，同时不暴露数据库、刷新凭据、设备私钥或 Gateway 令牌。成功意味着日常操作可以直接在页面完成，异常有明确去处，高风险动作有确认和审计，常规访问不依赖反复执行命令。

## Brand Personality

克制、可信、清晰。界面应像成熟的系统工具：安静但不空洞，信息充分但不拥挤，操作反馈直接，不用装饰掩盖状态。

## Anti-references

- 只展示数据却不能处理问题的只读看板。
- 依赖用户反复复制命令或临时链接才能进入的管理流程。
- 大面积空白、固定窄栏和低信息密度造成的“半成品后台”。
- 过度圆角、玻璃拟态、渐变堆叠、装饰性动画和模板化 SaaS 卡片墙。
- 用内部错误码、数据库字段或技术标识替代用户能理解的状态与说明。

## Design Principles

- 先给结论，再给证据：页面先说明是否需要处理，再按需展开技术细节。
- 状态必须可行动：凡是提示异常、待审核或权限风险，都要提供清晰的处理入口。
- 安全不制造日常摩擦：首次授权可以严格，之后应能从固定 HTTPS 地址直接进入；新浏览器仍需显式授权。
- 来源贯穿全程：记忆、审核和活动记录都要能回答“来自哪台设备、哪个 Agent、何时发生”。
- 高风险操作可恢复、可追踪：不在页面提供删除捷径；撤销和权限变更必须确认、校验版本并留下审计记录。

## Accessibility & Inclusion

以 WCAG 2.2 AA 为实现目标。所有核心操作支持键盘和清晰焦点，状态不只依赖颜色；表格在缩放和窄屏下仍可读，动画遵循 `prefers-reduced-motion`，错误信息说明原因和恢复方法。
