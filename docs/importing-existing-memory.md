# 导入已有记忆

已有的 `MEMORY.md`、`USER.md`、项目笔记和本机记录可以帮助新的共享库更快进入可用状态。不过它们形成的背景不同，可信度也不同。导入时应先把它们当作待审核材料，再决定哪些信息适合共享。

## 导入前先整理材料

先把来源分开保存，避免把完整会话、临时日志和密码文件一起送入扫描器。适合优先处理的内容包括已经确认的偏好、项目决定、设备事实和长期有效的工作约定。

| 来源 | 常见归属 | 导入时的注意点 |
|---|---|---|
| `USER.md` | `user` | 只保留用户已经确认且长期有效的偏好。 |
| `MEMORY.md` | `workspace` 或 `agent` | 先区分项目共识和某个 Agent 的私有经验。 |
| `SOUL.md` | `agent` 或 `private` | 不把角色设定自动扩大为工作区知识。 |
| 本机路径、端口、硬件记录 | `device` | 只让需要这台设备的 Agent 读取。 |
| 项目架构决定 | `workspace` | 附上来源和确认时间，便于后来追溯。 |
| 旧任务状态 | `session` 或 `archived` | 过期状态先保留为历史，不当作当前事实。 |

## 处理过程

```text
本地已有资料
  -> 扫描并生成预览
  -> 敏感信息筛查
  -> 分类、分块和作用域判断
  -> 去重与冲突检查
  -> 人工审核
  -> 写入共享库
```

每条材料应保留来源路径、内容哈希和批次号。审核记录需要能说明“这条信息从哪里来、谁确认、后来是否被替代”。这样在发现错误时可以定位并撤销对应批次，而不必猜测影响范围。

## 先生成预览

当前可用的第一步是扫描。它只读取指定目录并生成 JSONL 预览，不会直接把内容写入共享库：

```powershell
memory-import scan --source .\memory-folder --batch import_2026_07_03
```

预览文件应留在本机受保护目录。检查其中是否出现密码、令牌、私钥、连接串、内部地址、无关会话内容或过期状态；确认后再安排下一步审核和写入。

## 审核时重点看什么

- 内容是否已经由用户、项目文档或可信来源确认。
- 作用域是否足够小：设备信息不应变成所有工作区都能读取的知识。
- 是否与现有事实冲突，冲突时应保留双方来源并交由审核处理。
- 是否包含可识别的凭据、私密路径或命令式内容。
- 是否需要设置有效期、归档状态或后续复核时间。

批量导入的写入、回滚和结晶重建会继续围绕预览和审核流程提供，避免旧资料未经检查就影响新的协作。
